# Anomalies Endpoint Fix

## Issue Found ✅

The anomalies endpoint was returning 500 Internal Server Error due to a **parameter name mismatch**.

### Root Cause

**File**: `src/api_server.py`  
**Line**: ~708

The API endpoint was calling:
```python
anomalies = event_store.detect_anomalies(
    store_id=store_id,
    time_window_hours=time_window  # ❌ Wrong parameter name!
)
```

But the method signature in `src/event_store.py` is:
```python
def detect_anomalies(
    self,
    store_id: str,
    time_window: int = 24  # ✅ Correct parameter name
) -> List['Anomaly']:
```

This caused a `TypeError: detect_anomalies() got an unexpected keyword argument 'time_window_hours'`

---

## Fix Applied ✅

**Changed**:
```python
# Before (wrong)
anomalies = event_store.detect_anomalies(
    store_id=store_id,
    time_window_hours=time_window
)

# After (correct)
anomalies = event_store.detect_anomalies(
    store_id=store_id,
    time_window=time_window
)
```

## Anomaly Detection Types

The endpoint detects 4 types of anomalies:

1. **Sudden Crowd Surge**
   - Detects when current occupancy exceeds normal levels (>2 std dev)
   - Severity: Medium or High
   - Example: "Occupancy (50) exceeds normal levels"

2. **High Queue Abandonment**
   - Detects when abandon rate is unusually high compared to historical average
   - Severity: Medium or High
   - Example: "Queue abandonment rate (35.00%) is unusually high"

3. **Unusual Dwell Time**
   - Detects when dwell duration is outside normal range
   - Severity: Low
   - Example: "Dwell time (180.5s) is outside normal range"

4. **Off-Hours Activity**
   - Detects suspicious entries outside business hours (before 9 AM or after 9 PM)
   - Severity: Medium or High
   - Example: "Detected 15 entries outside business hours (9 AM - 9 PM)"
